JS Client

It's just another library of browser (you know, web) -oriented scripts and classes.

  • Basics

    Some of basic stuff that is just useful itself but required for heaps of other stuff contained in 'jsclient' library.
    • debug - crazily usable thing. Creates minimized debug console for scripts developers to be able to write logs (debugging information) and be able to open this console and investigate those logs in run-time.
    • JSON - it is a common way to encode / decode (materialize / serialize one can say) java-script objects.
    • require - for script to be able to access some classes, other scripts, report data, etc. from the server.

  • Effects

    Effects are a set of classes intended to provide some kind of visual effect (feedback). Those effects are attached to surfaces (usually it is a DIV element). Effect is displayed assuming that effect's intended view-port is the whole surface it is attached to. Every effect instance provides set of methods to control effect displayed.
    • Busy

  • Elements

    Elements add some extended document elements.
    • Dialog - Just kind of pop-up (dialogs)
    • ScalableImage - Image respecting container size

  • Features

    Features add some abilities to objects.
    • Deferred change (delayed onchange and such events)
    • Mouse Capture (drag scroll)
    • PopHint - An extended tooltip

  • Inputs

    Inputs are a set of classes representing inputs for user interaction. They are either attached, either automatically create real HTML input elements (such as: "INPUT", "TEXTAREA", "SELECT"...). It is designed this way to:
     1) virtually represent real DOM elements for third-party scripts compatibility;
     2) have the ability to wrap real HTML form to enhance simple HTML.
    • BoxSelect (single option / multiple options)
    • DropSelect (single option)

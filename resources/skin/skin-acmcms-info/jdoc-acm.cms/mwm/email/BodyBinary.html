    public class BodyBinary
extends BodyEncoded
    This is the base class for all mail bodies containing a binary content which must be encoded before transport. This could be attachments, images or other binary files.

    The class uses a Encoder object to prepare the body for transport. The Encoder is an abstract skeleton which developers can subclass to provide their own custom transport encodings.

    This body part contains a header object which manages the header fields. By default the body attempts to set the "Content-Transfer-Encoding" header field automatically. The value of this field is fetched from the Encoder object's getContextName method.

    Since the body of this class must be encoded before transport, the encoding is by default performed in the write method so an Encoder object must be provided before attempts to write the content. If the Encoder object is missing the class will throw an IOException.

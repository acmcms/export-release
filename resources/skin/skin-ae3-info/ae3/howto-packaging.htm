Packages do generally consist of executable code, some data and several service registration files. There are 'resource' and 'settings' folders allowed in the root of ae3 package, so all package contents must be located in either of those two folders.

The executable code can be either in form of CommonJS java script library or/and in form of Java jar file. CommonJS libraries should be located in 'resource/lib' hierarchy in form of 'library_name/jsApi_name.js' related internal scripts and data may be located in the same folder or it's sub-folders (for example: 'ru.myx.farm.dns' library with 'DnsWebService.js' web service and other files related to implementation of given service should be located in 'resource/lib/ru.myx.farm.dns/' folder.

The data may contain images, manuals, etc. Those files, depending on their relation to the whole package, to package libraries should be located either in package's folder, either in CommonJS library folder, either in some cases the location could depend on an API requirement, like 'resource/skin/' folder contain all the skins.

Service registration files are called descriptors and are used to specify properties to extensible interfaces provided by ae3 system and other packages. They are JSON files and only those descriptor files are allowed to be present at 'settings' root folder of any package.

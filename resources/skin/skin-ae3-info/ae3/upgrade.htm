Changes you need to perform on your existing ACMCMS sites in order to make sure that everything sill works as you expected.

  • 678 -> 679

    Changes:
    1. New default MIME types added.
    2. Directory structure for global skins in %protected% and %private% is changed to reflect directory structure for global public default skins bundled with system.
    3. Dropped support for obsolete alternative XML fieldset declaration.
    Must do:
    1. Ancient '<group...' fieldset declaration is not supported anymore, make sure all fieldsets are declared in a recommended way.
    2. Extra skins added to %protected%/skin/ directory must be moved to %protected%/resources/skin/ directory.
    3. Extra skins added to %private%/skin/ directory must be moved to %private%/resources/skin/ directory.

  • 677 -> 678

    Must do:
    1. For every DLS client registered in your site's config.xml file table d1ChangeLog needs to be dropped from database.
    2. If you use -Dserve.root java parameter while launching ACMCMS, you need to change it to -Dru.myx.ae1.path.web.
    3. Obsolete JavaScript array declarations like a = (1, 2, 3) are not supported anymore and must be replaced with a = [1, 2, 3].
    4. In all files you need to use global search/replace to get rid of 'Preffix' (misspelled) to 'Prefix', in past there was support for both notations in system's APIs.
    5. Make sure that all response filters in your types are starting with something like if( !Object.isMap( content ) ) return content;, cause before this version only map-like responses were forwarded through your response post-processing filters.

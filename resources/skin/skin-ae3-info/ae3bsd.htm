  • General

    It's an embedded-like software solution for running AE3 environment on i386 or x86-64 architectures using free and open-source FreeBSD operating system.

    While configuring operating system and choosing default parameters 'ae3bsd' assumes that the host is dedicated for the task.

    As a embedded database it is using is PostgreSQL, it is a very robust, modern, free and open-source relational database management system.

    Particular projects could still use external database connections as additional or primary connections.

    Some features of 'ae3bsd' solution:
    • automatic firewall setup
    • automatic embedded db configuration
    • soft-restart for group instances
    • automatic maintenance
    • simple consolidated configuration and management

  • Installation

    Provided that you already have installed FreeBSD system and logged in as a 'root' user before you proceed.

    1. Make sure you are running as 'root'. Use use 'man su' or 'man sudo' if you have any questions on this.
    2. Run the following command to install AE3: pkg_add http://myx.ru/distro/ae3/ae3.tbz
    3. reboot: reboot Wait until system restarts and re-connect to you console/terminal.
    4. now the script is installed and the last thing we could consider to be a part of 'installation' is to set an email address for system messages: ae3 config/email user@example.net you can specify several addressed at once, use syntax like this: ae3 config/email user@example.net,user@example.net

    At this point your system will have ae3 daemon installed and configured to be started automatically when operation system starts. This daemon would be running under newly-created 'ae3' user. In your 'bin' directory an 'ae3' command will be created with means to provide command line interface for various configuration and administration tasks.

  • Configuration and Operation

    'ae3' script acts like single command=line interface for most of tasks you will ever need to perform on you ae3 server. There is a robust set of simple commands provided by 'ae3':

    • Configuration

      defining and changing runnning system parameters.
      • ae3 config/email or ae3 config/email <email@addre.ss,more@somepla.ce> Internal administrative email recipient address could be examined or redefined by this command. You need to be 'root' to execute this command. ae3 config/email clear Keyword 'clear' could be used remove all email addresses.
      • ae3 config/logging or ae3 config/logging <LEVEL> Logging level (verbosity) could be examined or redefined by this command. Logging level must be one of: DEBUG, NORMAL or MININAL. You need to be 'root' to execute this command.
      • ae3 config/optimize or ae3 config/optimize <GOAL> Optimization mode (or goal) could be examined or redefined by this command. Optimization goal must be one of: DEFAULT, SIZE or SPEED. You need to be 'root' to execute this command.
      • ae3 config/ram or ae3 config/ram <RAM> ae3 deamon's JVM memory allocation could be examined or redefined by this command. Memory amount specification can be one of: 512m, 2g or any other specification in the same format. You need to be 'root' to execute this command.

    • Setting-Up

      pre-configuring some one-time aspects of the system.
      • ae3 setup/console configure current console to some pre-defined default settings. Any user can execute this command.
      • ae3 setup/host configure operating system as it was configured during 'ae3' installation. You need to be 'root' to execute this command.
      • ae3 setup/operator <user_name> grant 'operator' priveleges to a specified user. You need to be 'root' to execute this command.

    • Usage

      just normal everyday commands.
      • Connect to console: ae3 cli This will connect you to ae3 console session. You need to be 'operator' to execute this command.
      • Connect to console: ae3 console This will connect you to ae3 console session. You need to be 'operator' to execute this command.

  • Administration and Maintenance

    It is supposed that 'ae3' script should be able to solve all types of tasks associated with server maintenance.

    Configuration modification is available through a robust set of simple commands provided by 'acmbsd' script for 'root' user.

    • life-cycle

      some tasks related to long running maintenance.
      • Upgrade 'ae3' system ae3 upgrade It will upgrade code 'ae3' data and packages. You need to be 'root' to execute this command.
      • Re-run installation procedure. ae3 reinstall Execute all 'ae3' package installation scripts. Could be useful to restore environment if something went messy. You need to be 'root' to execute this command.
      • Restart daemon. ae3 restart This will re-start running daemon service as in 'stop and start'. You need to be 'root' to execute this command.
      • Start daemon. ae3 start This will start running daemon service if not running. You need to be 'root' to execute this command.
      • Stop daemon. ae3 stop This will stop running daemon service if running. You need to be 'root' to execute this command.
      • Watch logs. ae3 watch/log Connect and watch log messages on 'ae3' daemon logs. Those messages will appear in real time. Could be useful for debugging. You need to be 'root' or 'operator' to execute this command.
